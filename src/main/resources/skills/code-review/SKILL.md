# 代码审查技能

当用户请求代码审查时，按以下流程执行：

## 操作流程

1. **了解结构**：先用 `list_files` 工具了解项目目录结构与模块划分
2. **读取代码**：用 `read_file` 工具读取待审查的代码文件
3. **多维度分析**：从以下维度逐项分析
4. **输出报告**：给出结构化的审查报告

## 审查维度

### 1. 代码规范与可读性
- 命名是否清晰、一致（变量、函数、类）
- 是否符合语言社区风格规范
- 注释是否充分且准确

### 2. 潜在 Bug 与边界情况
- 空指针、越界、除零等常见异常
- 并发安全问题（竞态、死锁）
- 资源泄漏（未关闭的流、连接）
- 边界条件处理（空集合、负数、极大值）

### 3. 性能与安全
- 是否存在明显的性能瓶颈（N+1 查询、不必要的循环）
- 是否有注入风险（SQL、XSS、命令注入）
- 敏感信息是否妥善处理（日志脱敏、密钥管理）

### 4. 可维护性与扩展性
- 单一职责：函数/类是否承担过多职责
- 耦合度：模块间依赖是否合理
- 是否易于扩展（开闭原则）

## 输出格式

```
## 审查报告
- 严重问题：...
- 改进建议：...
- 优秀实践：...
```

给出具体的代码示例和修改建议，而非泛泛而谈。
